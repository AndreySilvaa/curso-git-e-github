O que é Git?

O Git é um sistema que grava as mudanças feitas em nossos arquivos ao longo do tempo por meio de "snapshots"

Ele te permite voltar em uma determinada mudança no código a qualquer momento

Permite a colaboração de vários desenvolvedores no projeto

Fluxo Git

+ git init: Inicializa o repositório Git (necessário)

+ git add _file_ : prepara um arquivo pra ser commitado commitados

+ git commit -m "mensagem": Faz o commit dos arquivos preparados. Na "mensagem" você pode colocar as alterações que foram feitas no código

Commit: É basicamente quando se é tirado um "print" do seu arquivo para guardar a versão dele

Comandos

+ git status: Mostra os "Untracked files" que são os arquivos que não estão preparados para commits mas que possuem modificações. Para prepará-los para commit use o comando "git add"

+ git rm --cached _file_: Desmonta a prepraração do arquivo para commit

+ git add .: Preprara TODOS os arquivos para serem commitados

+ git log: Serve para ver todos os seus commits

+ git branch: Visualiza os branchs

+ git branch _nome_do_novo_branch : cria um novo branch

+ git checkout _branch: Muda para o branch especificado

+ git merge _branch : Faz uma junção de branchs

+ git clone _url_do_repositorio: Clona um repositório remoto na sua máquina ( feito por meio do git bash dentro da pasta onde vai ficar o clone)

Arquivos que você não quer que sejam commitados

Quando você estiver trabalhando em algum projeto e for querer commitar os seus arquivos, talvez tenha um ou outro arquivo que você não queira que seja commitado, seja por possuir informações pessoais, delicadas ou qualquer outra coisa do gênero. Para solucionar esse problema você deve criar um arquivo chamado ".gitignore" e nele adicionar os arquivos que você quer que sejam ignorados quando você for fazer um commit

Entendendo sobre branch

Basicamente branch são pastas de arquivos. No mundo coorporativo, por exemplo, existem dois ambientes, um onde os códigos são criados, testados, atualizados, corrigidos, etc, e outro onde o aplicativo ou programa roda e funciona normalmente para os clientes. Nesses dois ambientes podemos dizer que cada um é um branch

Basicamente o github é uma plataforma que vai armazenar todos os seus projetos na nuvem, ou seja, ele vai criar um repositório remoto

Criando um repositório remoto

  1. No próprio github vá em new repository e crie o seu novo repositório colocando o nome dele
  2. Agora você precisa trazer o seu repositório local(Ja precisa ter criado) para o remoto e isso vai ser feito por alguns comandos simples no terminal do git no vscode
    • 1- git remote add origin https://github.com/AndreySilvaa/curso-git-e-github.git: Adiciona um repositório remoto com o nome origin
    • 2- git branch -M main: Muda o nome do branch de produção de master para main (é uma mudança que o github tá fazendo)
    • 3- git push -u origin main: Manda todos os seus arquivos e pastas do seu repositório local, para o seu repositório remoto origin, no branch main.

      + Para você dar esse comando push você deve estar no branch que você quer enviar

Clonando um repositório

  • Na pasta que você quer colocar o clone, abra o git bash nela(terminal do git)
  • Coloque o comando: git clone _url_do_repositorio

Atualizando um repositório local (git clone)

Para fazer atualizações no seu repositório local você pode utilizar os seguintes comandos:

  • git pull: Vai atualizar todos os branchs, todos os arquivos, pastas, etc, enfim, tudo, para igual ao que está no repositório remoto.(No exemplo do video essa atualização foi feita no branch main)
  • git pull origin _branch: Atualiza apenas um branch especifico (No exemplo do video foi feito no branch em que se desejava atualizar)
